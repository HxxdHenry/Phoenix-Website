Project: Gesture-Driven Desktop Control Panel

Task Description:

Create a floating on-screen control panel operated entirely by hand gestures:

  • Pinch to select control
  • Vertical finger movement to adjust value
  • Fist to confirm
  • Palm to cancel

Controls must include at least:

  • Scroll control
  • Zoom Control
  • One custom user-defined control

Mandatory Technical Requirements:

  • MediaPipe Hands
  • Landmark-based pinch detection
  • Temporal smoothing to avoid jitter
  • State machine design

Grading Rubric (20 Marks)

  • Gesture accuracy and stability: 6
  • Correct landmark logic explanation: 5
  • State handling and transitions: 5
  • Code structure and readability: 4

Automatic Fail Conditions:

  • Using frame-by-frame logic without state
  • Hardcoded screen coordinates
  • Copying logic

Project: Two-Hand Collaborative Gesture System

Task Description:

System must behave differently based on:

  • One hand detected
  • Two hands detected

Example Rules:

  • One hand draws
  • Two hands resize or rotate object

Mandatory Technical Requirements:

  • Multi-hand tracking
  • Hand ID consistency
  • Distance and angle-based logic

Grading Rubric (20 Marks):

  • Reliable hand separation: 6
  • Correct geometric reasoning: 6
  • No landmark confusion between hands: 4
  • Clean modular code: 4

Automatic Fail Conditions:

  • Treating two hands as one
  • Using only bounding boxes

Project: Proctoring Attention Monitor

Task Description:

  • Detect face presence
  • Track eye gaze direction
  • Flag distraction if user looks away continuously for more than N seconds

Mandatory Technical Requirements:

  • MediaPipe Face Mesh
  • Eye landmarks
  • Time-based buffering

Grading Rubric (20 Marks):

  • Eye landmark usage correctness: 6
  • Time logic implementation: 6
  • False positive reduction: 4
  • Code clarity: 4

Automatic Fail Conditions:

  • Checking only head direction
  • Single-frame decisions

Problem Statement:

In laboratories and workshops, physical interaction with keyboards, mice, and control panels can be unsafe due to contamination, gloves, hazardous materials, or hygiene restrictions. Your task is to design and build a touchless gesture-based control system that allows a user to operate laboratory or workshop equipment without touching any physical input device. The system must use a camera-based computer vision approach to detect hand gestures and convert them into safe, deliberate control actions. This is a single-round hackathon. You must design, implement, and explain your system within the given time.

What You Must Build (Mandatory):

  • Real-time hand detection using a camera
  • Minimum 3 distinct gestures
  • State-based system (IDLE, MENU, ACTION, CONFIRMATION)
  • Emergency stop gesture
  • Confirmation mechanism before executing actions
  • On-screen visual feedback

Keyboard or mouse input after the system starts is not allowed.

What You Must Explain:

  • Why each gesture was chosen
  • How accidental activation is prevented
  • Behavior under lag or poor lighting
  • Implemented vs simplified features

Submission Requirements:

  • Maximum 6-slide PPT
  • Live working prototype
  • Partial implementation allowed

Constraints:

  • Team size: 2-3 members
  • Programming language: Python
  • Computer vision must be used
  • No prerecorded demos
  • No future-work-only solutions

Evaluation Rule - Teams will be judged on:

  • System thinking and design
  • Gesture robustness
  • Safety logic
  • Demo reliability
  • Clarity of explanation

Disqualification Conditions:

  • No emergency stop gesture
  • No state-based logic
  • Keyboard or mouse used during demo
  • Inability to explain design choices
